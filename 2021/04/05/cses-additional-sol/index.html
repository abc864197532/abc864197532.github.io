1087 - Shortest Subsequence

連結

題解

一個直觀的greedy法就是從左掃到右，沿途計算目前四種字元的數量
若四種字元的出現過了，代表答案長度一定需要+1，否則就會出現subsequence
所以就把當前跑到的字元加到答案裡，並將數量歸零
跑完後要記得加上一個沒出現過的字元

1146 - Counting Bits

連結

題解

先分別計算第0個bit、第1個bit、…的答案，再加起來就好
觀察一下可以發現第iii個bit每2i+12^{i + 1}2i+1個數會循環，且每個循環節前2i2^i2i個數會是0，其他會是1
所以對每個bit直接算出循環的數量，再加一加就能得出答案了

1670 - Swap Game

連結

題解

遊戲最多只有9!種狀態，所以就蓋一張9!個點的新圖，將每種能互相到達的狀態用一條邊連接
然後跑一遍bfs即可

1134 - Prüfer Code

連結

題解

首先，沒被列在當前Prüfer Code裡的點一定是葉子
所以做法其實很單純，跑過一次陣列，每次挑出一個index最小的葉子跟陣列的點連邊
若該點已經不在Prüfer Code裡了就把他丟到葉子的set裡

梗題
把編號小的指到邊號大的，出來的圖就一定無環

2177 - Strongly Connected Edges

連結

題解

結論

  1. 圖不連通或是圖上有橋，則無解
  2. 否則蓋一棵dfs樹，把樹邊的方向定為parent to child，回邊訂為child to parent，這樣一定是對的

說明

圖上沒有橋 -> dfs樹每一條樹邊一定會被一條回邊蓋到 -> 必定可以從一個點走到根
走到根之後就能直接經由樹邊走到所有點了

2179 - Even Outdegree Edges

連結

題解 Part1

先來考慮樹的case吧，dfs(v)會回傳將vvv的所有子樹的點(不包含vvv)變偶出度，vvv的出度是奇數還是偶數
這樣一來，對於一個點vvv，它的兒子uuu會有兩種case

  1. dfs(u) == 1，就將邊從uuu指向vvv
  2. dfs(u) == 0，就將邊從vvv指向uuu
    最後只要檢查dfs(root)回傳的是不是0即可
題解 Part2

再來一般圖的case，會發現前面的方法還是可行的，只要將回邊的方向全部定為parent to child即可
這樣就不會動到該點子樹的奇偶性

1142 - Advertisement

連結

題解

經典題
先計算出l[i]代表從最大的indexjjj s.t. h[j] < h[i]
在算出r[i]代表最小的indexjjj s.t. h[i] > h[j]
這個可以用stack做單調對列做完，做法如下：
首先我們觀察到若j<i,h[j]≥h[i]j < i, h[j] \geq h[i]j<i,h[j]≥h[i]，則jjj一定不會是之後的答案(因為jjj會先被iii擋掉)
所以我們可以開一個stack存下還有可能作為答案的index
每當加入一個數，就看它可以讓哪些位置不再作為答案，把它一個一個pop掉
結束之後就直接取stack的top，它就會是我們要的l[i]了

算完之後再算h[i]∗(r[i]−l[i]+1)h[i] * (r[i] - l[i] + 1)h[i]∗(r[i]−l[i]+1)的最大值就會是答案惹

2186 - Special Substrings

連結

Hint

Hash

題解

須滿足每個字母的hash值總和為0

這樣一來問題就轉換成有幾個區間和為0

2229 - Permutation Inversions

連結

題解

dp[n][k]代表長度為nnn、有kkk個inversion的permutation數量
則轉移式可考慮窮舉nnn的位置：
dp[n][k]=∑0≤i≤n−1dp[n−1][k−i]dp[n][k] = \sum_{0 \leq i \leq n - 1}{dp[n - 1][k - i]}dp[n][k]=∑0≤i≤n−1​dp[n−1][k−i]
其中iii為增加的inversion數量

一段區間和，可用前綴和加速，複雜度O(nk)\mathcal{O}(nk)O(nk)

1655 - Maximum Xor Subarray

連結

題解

首先先套前綴和，答案等於pre[r]⊕pre[l]pre[r] \oplus pre[l]pre[r]⊕pre[l]的最大值

首先先把所有前綴值建一棵0-1trie
接下來要query一個數跟trie上所有數的最大xor值
由於二進位的性質，最高位可以先放1就放1是最好的
所以可以在那棵trie上走，遇到0可以往1就往1走，反之亦然
之後取全部最大值即可

1664 - Movie Festival Queries

連結

Hint

倍增

題解

超級喜歡這題> < 不過有點難想

來考慮一下倍增吧
dp[i][j]代表從時間iii開始，看2j2^j2j個movie的最短所需時間
倍增的轉移：dp[i][j]=dp[dp[i][j−1]][j−1]dp[i][j] = dp[dp[i][j - 1]][j - 1]dp[i][j]=dp[dp[i][j−1]][j−1]

之後每次query就從最大的bit開始看，看跳完會不會超過限制
如果不會超過就更新答案，繼續往後跳

1702 - Tree Traversals

連結

題解

preorder: (根)(左子樹)(右子樹)
inorder: (左子樹)(根)(右子樹)

做法就是：

  1. preorder的開頭是根，在inorder找到根所在的位置
  2. 現在知道左子樹在preorder和inorder的區間了，所以遞迴下去
  3. 同樣道理，遞迴右子樹的區間
  4. 輸出根

1704 - Network Renovation

連結

題解

結論

令葉子數量為mmm，則答案為⌈m2⌉\lceil \frac{m}{2} \rceil⌈2m​⌉
連邊的方法為將葉子按照以不為葉子的點為根做dfs的時間戳記排序好，將第iii個葉子連到第⌊m2⌋+i\lfloor \frac{m}{2} \rfloor + i⌊2m​⌋+i個葉子
若mmm為奇數，再將第mmm個葉子連到第111個葉子即可

證明

首先，由於每一條邊最多只會讓兩個葉子滿足條件，所以答案會有下界為⌈m2⌉\lceil \frac{m}{2} \rceil⌈2m​⌉
接下來就只要證明這樣的連邊方法是好的

首先，對於每一條邊，可以簡化成以下的樣子

設BBB為位於1子樹的葉子集合，大小為bbb
AAA為在那bbb個點之前的葉子集合，大小為aaa
CCC為在那bbb個點之後的葉子集合，大小為ccc
所以現在要證明的就是除非a = 0 & c = 0，否則至少有一條從AAA到BBB的邊或從BBB到CCC的邊

WLOG, a>0a > 0a>0
定義a0,b0,c0a0, b0, c0a0,b0,c0為第一個AAA的點，第一個BBB的點和第一個CCC的點(c0c0c0不一定存在)
則以下分為三種case:

  1. a0a0a0連到集合AAA的點，則與b0b0b0連邊的點必屬於AAA集合
  2. a0a0a0連到集合BBB的點
  3. a0a0a0連到集合CCC的點，代表c>0c > 0c>0，c0c0c0存在，則與c0c0c0連邊的點必屬於BBB集合

所以對於任意一條邊，必會被一條新建的邊覆蓋到，也就代表該作法成立

1707 - Graph Girth

連結

題解

對於每個點，我們計算出經過他的最小環，這樣再把全部取最小值即可
而求出最小環的方法就是跑一次bfs，若遇到已經跑過的點就代表環產生，就直接更新最小值
這個做法是好的原因是在本題的限制下，O(n(n+m))\mathcal{O}(n(n + m))O(n(n+m))不會爆
